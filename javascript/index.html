Arguments

You can pass in a variable number of arguments to a function, how do you get them all?


function sum() {
  var sum = 0;
  for (var i=0; i < arguments.length; i++) {
    sum += arguments[i];
  }
  return sum;
}

Functional Scoping

JavaScript has functional scoping vs block scoping. Variables defined within a function, are accessible anywhere in that function, but are not accessible outside of that function.

Functional Scoping


function myFunk() {
  var a = 2;
  function innerFunk() {
    a += 2;
  }
  console.log(a); // 2
  innerFunk();
  console.log(a); // 4
}
console.log(a); // undefined

What About?


function myFunk() {
  var a = 2;
  function innerFunk() {
    var a = 4;
  }
  console.log(a); // 2
  innerFunk();
  console.log(a); // 2, why?
}
console.log(a); // undefined

Closure

A Closure in JavaScript is when a scope "survives" even after a function has executed.

Closure


var myAdder = (function() {
  var count = 0;
  return {
    add: function(x) {
      count += x;
      return count;
    }
  }
}());
myAdder.count; // undefined
myAdder.add(3); // 3
myAdder.add(10); // 13

Constructor Functions

Constructor Functions

JavaScript does not have classes, but there is an object making syntax that looks similar to creating a class.

Constructor Functions


function Person(firstName, lastName) {
  this.name = firstName + ' ' + lastName;
}
var preet = new Person('Preet', 'Jassi');
preet.name; // 'Preet Jassi'

Constructor Functions

Using the new operator creates a new object and executes the constructor function. Anything that is bound using this in the constructor function is now bound to that new object.

This

This

This is a reserved keyword that refers to an object. What that object is depends upon how the function is executed.

As a method in an object


var myObj = {
  myVal: 20,
  myMethod: function() {
    return this.myVal;
  }
};
myObj.myMethod(); //20

In a Constructor Function


function Person() {
  this.firstName = 'Preet';
  this.greet = function() {
    return 'Hello, ' + this.firstName;
  }
}

In an invokable function


function adder(a, b, c) {
  this.val = a + b + c;
}
adder(1, 2, 3);
alert(val); // 6

Inheritance

Prototype Chain

All objects have a hidden link, called a prototype, that links to other objects. When you access a member on that object, it checks to see if it is defined on that object itself. If it is not defined on that object, it looks at the object at the other end of that hidden link to see if it is defined there. If the member is defined on the objects prototype, it is returned.

This is called the "prototype chain".

Prototype


function Person(firstName, lastName) {
  this.name = firstName + ' ' + lastName;
}
var preet = new Person('Preet', 'Jassi');
preet.name; // 'Preet Jassi'
Person.prototype.salut = function() {
  alert('Bonjour ' + this.name + '!');
}
var bob = new Person('Bob', 'Marley');
bob.salut(); // this will alert Bonjour Bob Marley!
preet.salut(); // this will alert Bonjour Preet Jassi!

Inheritence

JavaScript does not have inheritance, but we can do something clever with the prototype to inherit behaviour from one object to another.

Prototype


function Person() {
  this.setName = function(firstName, lastName) {
    this.name = firstName + ' ' + lastName;
  };
}
function Female() {
  this.gender = 'F';
  Person.call(this);
}
Female.prototype = new Person();
Female.prototype.constructor = Female;

var mary = new Female();
mary.setName('Mary', 'Ann');
mary.name; // 'Mary Ann'
